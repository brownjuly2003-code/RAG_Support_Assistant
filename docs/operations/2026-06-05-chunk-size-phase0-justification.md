# chunk_size=800/200 ОБОСНОВАН — Phase 0 co-occur гейт закрыл вопрос без production-sweep

- Дата: 2026-06-05.
- План: `docs/plans/2026-06-05-chunk-size-justification.md`. Exit-критерий Phase 0:
  «cap-кандидат не возвращает связок → 800 остаётся, sweep не нужен».
- Метод: co-occur методика residual-miss диагноза — для каждого из 100
  curated-кейсов проверяется, живёт ли полный kw-набор внутри хотя бы одного
  чанка; нарезки воспроизведены локально `structural_split(cap, 200)`,
  без моделей.

## Результат

| cap | чанков | связок живо | возвращено vs 800 | потеряно |
|---|---|---|---|---|
| 800 (текущий) | 5589 | 98/100 | — | — |
| 1200 | 4798 | 99/100 | 1 (cargo-loss-required-fields) | 0 |
| 1600 | 4621 | 99/100 | та же 1 | 0 |

## Вывод — production-sweep НЕ нужен

1. Единственная возвращаемая связка — `cargo-loss-required-fields` — в
   production-стеке **уже FULL** (закрывается объединением top-5 чанков ещё с
   плеча C; перепроверено на D2-кандидатах). Sweep гнался бы за кейсом,
   который не сломан.
2. Вторая neither-связка (`subject-rights-third-party-masking`) не оживает
   даже при cap=1600 — kws разнесены дальше соседних секций; это не аргумент
   за cap, рычаг — parent-expansion window / query-side.
3. Остаточный MISS текущего стека (`customs-clearance-fields`) — связка ЖИВЁТ
   в одном чанке при cap=800 (пул C, r27): проблема ранжирования, не нарезки.
4. Потерь от большего cap нет (0 lost), но и выигрыша нет: parent-expansion
   (default ON c `f82f262`) уже склеивает соседние секции post-rerank, покрывая
   механизм «узкая секция потеряла контекст» дешевле, чем пере-ингест с новым cap.

## Phase 2 (overlap-чувствительность) — SKIPPED по критерию плана

Затронутых кейсов < 10 (фактически 1, и тот не сломан) → фаза скипнута,
как предписывает план.

## Зафиксированное обоснование 800/200 для этого корпуса

- cap=800 режет 18.3% секций (медиана секции 563 chars — 82% секций cap не
  трогает вовсе);
- из порезанных НИ ОДНА не уносит kw-связку, которой нет при большем cap,
  кроме одного уже-закрытого кейса;
- слабое место дефолта (узкие секции без соседнего контекста) закрыто
  parent-expansion'ом, измеренным отдельно (FULL 96/100).

Пересмотр вопроса оправдан только при смене корпуса (другая структура секций —
повторить замер распределения, 5 минут локально) или при появлении новых
MISS-кейсов класса «связка разрезана» в eval.
